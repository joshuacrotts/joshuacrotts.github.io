Problem Set 3 - Standard & Tail Recursion

Assigned: January 28, 2026

Due: February 4, 2026 at 11:59 PM EST

Objectives

  • Students learn to design more complex methods.
  • Students understand and describe the differences between standard recursion and tail recursion.
  • Students design methods that call private helper methods to solve a problem.

Instructions

For each problem, create a class named ProblemX, where X is the problem number (e.g., Problem1.java). Write corresponding JUnit test classes named ProblemXTest. Each method must include proper Javadoc comments describing the (1) purpose of the method, (2) parameters, and (3) return value.

For problems containing multiple parts, write the methods inside the same class.

Do not round your solutions. For example, do not use Math.round or float your output to, e.g., 2 decimal places, unless specified.

What You Cannot Use

You may not use anything beyond Chapter 2.2. This includes but is not limited to:

  • Loops
  • Arrays or data structures
  • Regular expressions

Any violation results in a score of 0 on the problem set.

Please contact a staff member if you are unsure as to whether you're allowed to use something.

Problems

  1. Design the boolean isPalindromeTR(String s) tail recursive method, which receives a string and determines if it is a palindrome. Recall that a palindrome is a string that is the same backwards as it is forwards (e.g., "racecar"). Do not use a (character) array, StringBuilder, StringBuffer, or similar, to solve this problem. It must be naturally recursive.

    Files to Submit: Problem1.java, Problem1Test.java

  2. This problem has two parts.

    1. The hyperfactorial of a number, namely \(H(n)\), is the value of \(1^1 \cdot 2^2 \cdot \dots \cdot n^n\). Because the resulting values are outrageously large, use the long datatype. Design the standard recursive long hyperfactorial(long n) method, which receives a long integer \(n\) and returns its hyperfactorial.

    2. Then, design the long hyperfactorialTR(long n) method that uses tail recursion and accumulators to solve the problem. Hint: you will need to design a private static helper method.

    Files to Submit: Problem2.java, Problem2Test.java

  3. This problem has two parts.

    1. The subfactorial of a number, namely \(!n\), is the number of permutations of \(n\) objects such that no object appears in its natural spot. For example, \(!3 = 2\). We define subfactorial recursively as:

      \[ \begin{aligned} !0 &= 1 \\ !1 &= 0 \\ !n &= (n-1)\cdot\big(!(n-1) + !(n-2)\big) \end{aligned} \]

      Design the standard recursive long subfactorial(long n) method, which receives a long integer \(n\) and returns its subfactorial. We once again use the long datatype instead of int to handle larger values.

    2. Then, design the long subfactorialTR(long n) method that uses tail recursion and accumulators to solve the problem. Hint: you will need to design a private static helper method.

    Files to Submit: Problem3.java, Problem3Test.java

  4. This problem has two parts.

    1. Design the standard recursive String collatz(int n) method, which receives a positive integer and returns the Collatz sequence, as a string, for that integer:

      \[ \begin{aligned} \texttt{collatz(1)} &= \texttt{1} \\ \texttt{collatz(n)} &= \texttt{collatz(3 * n + 1)} \text{ if }\texttt{n}\text{ is odd} \\ \texttt{collatz(n)} &= \texttt{collatz(n / 2)} \text{ if }\texttt{n}\text{ is even} \end{aligned} \]

      The sequence continues until it reaches 1. For example, collatz(5) returns "5,16,8,4,2,1". The last number cannot have a comma afterwards.

    2. Then, design the String collatzTR(int n) method that uses tail recursion and accumulators. Hint: you will need to design a private static helper method.

    Files to Submit: Problem4.java, Problem4Test.java

  5. This problem has two parts.

    A parenthesized string is a string enclosed by parentheses. For example, "(abc)pqr(de)" contains two parenthesized strings: "abc" and "de".

    Assume there are no nested parentheses, all parentheses are balanced, and each parenthesized string contains at least one character.

    1. Design the standard recursive String collectParenthesizedStrings(String s) method, which returns a single string containing all parenthesized substrings.

    2. Design the String collectParenthesizedStringsTR(String s) method that solves the same problem using tail recursion. Remember to include the relevant access modifiers.

    Files to Submit: Problem5.java, Problem5Test.java

  6. Design the tail recursive char findLastNonDigitCharTR(String s) method that returns the last character in the string that is not a digit. If no such character exists, return the NUL character, i.e., '\0'. You will need to design a helper method. Remember to include the relevant access modifiers.

    Files to Submit: Problem6.java, Problem6Test.java
